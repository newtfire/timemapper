Looking at how TimeMapper runs and what they used to make it.
- in /public/js there are a number of js files that are used to put the program together 
- backend.gdocs.js is interesting to look at since that is where they store their functions to allow for sheets to work with timemapper
    - they seemed to use recline to read the sheet, but they also removed something called recline in public/vendor folder, im not sure if this is the same as the functions they were using but im noting it down
    - they are still using recline functions in the backend.gdoc.js file despite notr having it in their vendor folder
- leaflet isnt reading the sheet itself, the backend is handling everything
    - leaflet is being used to creat the icons for each location added as seen in view.js lines 257-323
    - they are also using geoJSON data for leaflet which i didnt mess with when i used the library
- TimeLineJS is being used to the timeline itself and there are functions in view.js that let the user click a point on the leaflet map that will update the timeline to the correct event
